On screen Keyboard

YouView On Sony TV

Brief

Create an on-screen search capable of replacing the remote control as an input device, ensuring that it would work across all YouView devices and products

Client

BT TV

Project Date

2019

Role

Lead Designer

The problem to solve

Goals

As shareholders were starting to invest more in the YouView product, we aligned roadmaps to create a unified approach across the main investors. With one eye on the future road map, we were instructed to investigate ways for a user to interact with the software without relying on the remote control. The first phase for this was an on-screen keyboard.

Secondary goals

Although the search was the main function of the onscreen keyboards, we also had to keep an eye on other areas of the interface where a keyboard would be used instead of physical buttons. One main area would be entering passwords, especially for internet connection set up. We would need to include an easy way for users to enter special character s which would not be common in normal search queries.

Why?

If we didnt move ahead with this project, we were seriously limiting our shareholders with their creative process. We knew there was a desire to remove letters from the physical remote control, and our insights told us that it would lead to confusion and irritation from our userbase.

We also opened up the ability for another way the user could interact with the box, in turn allowing new and improved features such as voice recognition input to be brought into the project.

I was the lead designer on this project, working alongside a wider design team as well as in house development and a mixture of external and internal QA testing team. This allowed us to explore and experiment with prototypes built directly into the set-top box using HTML5 technology, for fast iterations and tweaks.

What?

Competitor Research

As part of the research into this project, the UX&Design team came together to look at competitors and the different styles they had used for their own on-screen board. We made a conscious effort to not limit ourselves to only TV interfaces and had a wide range of examples from across the globe.

These examples were then critically reviewed for their pros and cons, approaching each example with a set of personas to try get the widest thought process and practicality.

These Pros/Cons were then grouped into parent features and then ranked in order of importance by the team. Together we were able to then come up a rough concept that we could present back to stakeholders which we were all aligned on.

Gathering Data

We approached this project with a keen goal of getting as much user feedback as possible. We launched an on screen dialogue asking for people who would be interested in joingin a user testing session to communicate with. THe approach for testing was to use two different versions of the keyboard that we had decided upon as a team.

Concepts

I designed a few variations and through selections and narrowsed down the design until we reached a common consensus for the design

Examples of animations for the button press interaction

AB Testing

I wanted to ensure any results we hypothesis would come into fruition so we needed to get the designs infront of as many live users as possible. Using a third party agency we recruited a number of people to be exposed to 2 different versions of the design.

Presenting ideas to stakeholders

As a team we had two solutions to the on screen keyboard problem. It was now time to present these solutions back to the wider audience of stakeholders and in turn the wider company for feedback.

Design Solution

The final designs were implremented into the youview set top box, with the highest quality CSS animations implemented on the higher powered BT 4K box.
